# Loom

> Make for AI workflows.

Loom is a Git-native workflow language for packaging reusable AI work as code.
It compiles parameterized `.loom` programs into agent-ready artifacts, traces, and deterministic tests.

> **v0.1.0 — deterministic developer preview.** Loom v0 makes **no LLM calls** and **no network calls**. It turns `.loom` source into files, a provider-neutral Program IR, and run traces using only your inputs and local files. See [Status](#status).

---

## Why

Feeding instructions to AI coding agents is currently a mess:

- **Copy-pasted prompts** scattered across wikis, Slack, and people's heads.
- **Manual placeholder replacement** ("remember to swap in the file name").
- **Duplicated `AGENTS.md` / Claude / Copilot / Cursor instructions** that drift out of sync across repos.
- **No import/export boundary** for sharing prompt logic between projects.
- **No deterministic tests** for prompt-generating workflows.
- **No trace** of what generated a given artifact, or from which inputs.

Loom treats this as a **build problem**. You write declarative `.loom` programs, compile them into artifacts, and get determinism, imports, tests, and traces for free. The practical v0 use case is AI-governance infrastructure for repos: generate agent instructions, review briefs, prompt libraries, and release checklists from one source of truth, then gate them in CI.

## What this looks like in practice

Use Loom when AI-facing files should be generated, not hand-maintained:

- generate `AGENTS.md` from a shared team prompt and repo-specific params;
- generate Copilot/Cursor/Claude instructions from the same source;
- generate PR review checklists for sensitive areas like auth, billing, or migrations;
- regenerate committed artifacts in CI and fail the build if they drift;
- keep a trace of the exact inputs that produced each artifact.

Loom is most useful for teams with more than one repo, more than one coding
agent, or more than one copy of the same prompt.

## Install

```bash
npm install -g loom-ai
```

The package is named **`loom-ai`** on npm, but the command is still:

```bash
loom
```

### From source (secondary option)

```bash
git clone https://github.com/glamcoder/loom-ai.git
cd loom-ai
npm install
npm run build   # bundles to dist/index.js (bin: loom)
npm link        # puts `loom` on your PATH
```

No global install? You can run the CLI straight from the repo with `npm run loom -- <args>` (note the `--`).

## 60-second quickstart

```bash
# 1. Validate a program (parse + resolve imports + type/effect check, no writes)
loom validate examples/refactor.loom

# 2. Compile a program to its provider-neutral IR (read-only, prints JSON)
loom compile examples/refactor.loom PrepareRefactor \
  --method calculateTotalCost --file src/billing/costs.ts

# 3. Run it to write artifacts to disk + record a trace
loom run examples/refactor.loom PrepareRefactor \
  --method calculateTotalCost --file src/billing/costs.ts

# 4. Run the workflow tests
loom test examples/refactor.loom
```

Expected output:

- `src/billing/AGENTS.md` — a rendered refactoring brief.
- `.loom/runs/<run-id>/trace.json` — a record of the run.
- `loom test` reports `1 passed, 0 failed`.

See the [Getting started guide](docs/getting-started.md) for a step-by-step walkthrough.

## Tiny example

```hcl
# hello.loom
module "workflows.hello" {
  version = "0.1.0"
}

prompt "Greeting" {
  param "name" {
    type    = Text
    default = "world"
  }

  returns = Markdown

  template = """
# Hello, {{ name }}

This file was generated by Loom.
"""
}

export program "Hello" {
  param "name" {
    type    = Text
    default = "world"
  }

  effects = ["fs.write"]

  step "render" {
    use  = Greeting
    with = { name = param.name }
  }

  step "write" {
    use  = fs.write
    with = {
      path    = "HELLO.md"
      content = step.render.output
    }
  }

  output "greeting" {
    type = Markdown
    from = step.render.output
  }
}

test "hello_greets_the_name" {
  program = Hello
  with    = { name = "Ada" }

  expect {
    output "greeting" contains "Hello, Ada"
    writes file "HELLO.md"
    effects = ["fs.write"]
  }
}
```

## What Loom is good for today

- Generate `AGENTS.md` for coding agents.
- Generate PR review instructions.
- Package reusable refactoring prompts.
- Create prompt libraries shared across repos.
- Run deterministic checks in CI.
- Trace generated AI workflow artifacts.

## What Loom is not

Loom v0 is a deterministic build/workflow layer for AI artifacts. It is:

- **not** LangGraph (no agent runtime / orchestration loop)
- **not** BAML (no typed LLM function calls — yet)
- **not** promptfoo (not an eval/benchmarking harness)
- **not** a hosted agent platform
- **not** an LLM API wrapper in v0 (it makes no model calls)

See [docs/comparisons.md](docs/comparisons.md) for a fair, detailed breakdown.

## Status

- **v0.1.0 developer preview.**
- Deterministic only — no LLM calls, no network calls.
- No provider SDKs.
- Package name: `loom-ai`. Command name: `loom`.
- **Loom is a working name** and may change before a stable release.

## Roadmap

- **v0** — deterministic artifact compiler _(current)_
- **v1** — `llm.complete` with a `MockLlmProvider` first
- **v2** — parse / validate helpers for model output
- **v3** — workflow control: retries, checks, conditions
- **v4** — ecosystem / packages

Directional and subject to change based on feedback. See [docs/roadmap.md](docs/roadmap.md).
