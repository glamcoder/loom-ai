examples/refactor.loom 
export program "PrepareRefactor" {
  param "method" { type = Symbol; required = true }
  param "file"   { type = Path; required = true }

  effects = ["fs.write"]

  step "render_instructions" {
    use  = refactor.RefactorMethod
    with = { method = param.method, file = param.file }
  }

  step "write_agent_file" {
    use  = fs.write
    with = {
      path    = dirname(param.file) + "/AGENTS.md"
      content = step.render_instructions.output
    }
  }
}

Why Loom exists

AI instructions deserve build-system discipline.

Copy-pasted prompts are hard to review, hard to reuse, and hard to trust. Loom treats prompt and artifact generation as source code: typed inputs, imports, effects, tests, and traces.

Before

Prompt logic drifts

Instructions hide in docs, Slack, editor configs, and old PRs. Each repo becomes a one-off snowflake.

Before

Placeholders are manual

Humans remember to swap `{{ method }}` and hope generated files still match their source.

After

Artifacts become reproducible

Loom compiles `.loom` source into files, IR, and traces that can be checked in, reviewed, and gated in CI.

How it works

Write, validate, run, verify.

Loom v0 is deliberately deterministic: no provider SDKs, no model calls, and no network calls. That makes it reliable enough for reviews and CI today.

1. Write `.loom`

Define prompt modules, programs, params, steps, effects, and tests.

2. Validate

Catch parse, import, type, reference, and effect errors early.

3. Run

Render prompts, write declared files, and emit trace metadata.

4. Test in CI

Run deterministic workflow tests with an in-memory filesystem.

60-second quickstart

Install the CLI, then compile your first workflow.

  • Install the npm package named `loom-ai`. The command you run after installing is `loom`.
  • Validate a module before running it. Validation writes nothing and reports clear coded diagnostics.
  • Compile to provider-neutral JSON when you want to inspect the execution plan in review.
  • Run tests in CI. They execute fully in memory, so they are deterministic and safe.
terminal from docs/getting-started.md 
npm install -g loom-ai

loom validate examples/refactor.loom

loom compile examples/refactor.loom PrepareRefactor \
  --method calculateTotalCost \
  --file src/billing/costs.ts

loom run examples/refactor.loom PrepareRefactor \
  --method calculateTotalCost \
  --file src/billing/costs.ts

loom test examples/refactor.loom

What ships in v0

A small language for repeatable AI work.

Modules

Local imports and exports

Split reusable prompts from workflows and import only exported symbols through relative paths.

Params

Typed inputs

Required params and defaults make placeholder substitution explicit and validated.

Effects

Auditable side effects

Programs must declare effects like `fs.write` before they can write files.

Tests

Deterministic assertions

Assert rendered outputs, expected file writes, and declared effects without touching disk.

Traces

Generated artifact history

Each run records inputs, steps, outputs, files written, and diagnostics in `.loom/runs`.

IR

Provider-neutral plans

`loom compile` prints JSON that future model providers can execute without changing source semantics.

Positioning

Not an agent runtime. Not an eval harness. A build layer.

Loom complements tools like BAML, LangGraph, DSPy, promptfoo, and coding agents by giving them clean, versioned, tested inputs.

BAML

BAML is for typed LLM functions inside your application. Loom v0 makes no model calls; it packages reusable AI artifacts and workflows.

LangGraph

LangGraph orchestrates stateful agents at runtime. Loom compiles deterministic artifacts and plans ahead of time.

promptfoo

promptfoo evaluates prompt and model behavior. Loom tests deterministic prompt generation and file artifacts.

Roadmap

Deterministic first, model-aware later.
