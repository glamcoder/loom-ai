# Tutorial: Build a prompt library

## What you'll build

A small **shared prompt library** and two workflows that compose it. You'll see exported prompts, a private helper prompt, importing with an alias, and reusing one library across multiple programs. This uses the shipped `examples/prompts/common.loom` and `examples/prompt-library.loom`.

## The source files

`examples/prompts/common.loom` — the shared library. `Header` and `Signoff` are exported; `Disclaimer` is a private helper that importers can't see:

```hcl
module "prompts.common" {
  version = "0.1.0"
}

export prompt "Header" {
  param "title" {
    type     = Text
    required = true
  }

  param "audience" {
    type    = Text
    default = "developers"
  }

  returns = Markdown

  template = """
# {{ title }}

Audience: {{ audience }}
"""
}

export prompt "Signoff" {
  param "owner" {
    type     = Text
    required = true
  }

  returns = Markdown

  template = """
---
Maintained by {{ owner }}.
"""
}

# Private helper prompt (not exported).
prompt "Disclaimer" {
  returns = Markdown

  template = """
> Generated by Loom AI. Edit the .loom source, not this file.
"""
}
```

`examples/prompt-library.loom` — composes the library into two programs. It imports `common` with an alias and also defines a module-local helper prompt (`Footer`) referenced without an alias:

```hcl
module "workflows.prompt_library" {
  version = "0.1.0"
}

import "./prompts/common.loom" as common

# A module-local, unexported helper prompt, referenced without an alias.
prompt "Footer" {
  param "owner" {
    type     = Text
    required = true
  }

  returns = Markdown

  template = """
_Questions? Ask {{ owner }}._
"""
}

export program "TeamGuide" {
  param "team" {
    type    = Text
    default = "Platform"
  }

  param "owner" {
    type    = Text
    default = "the Platform team"
  }

  effects = ["fs.write"]

  step "render_header" {
    use  = common.Header
    with = {
      title    = "Team guide"
      audience = param.team
    }
  }

  step "render_footer" {
    use  = Footer
    with = { owner = param.owner }
  }

  step "write_guide" {
    use  = fs.write
    with = {
      path    = "GUIDE.md"
      content = step.render_header.output + step.render_footer.output
    }
  }

  output "guide" {
    type = Markdown
    from = step.render_header.output + step.render_footer.output
  }
}

export program "OnboardingDoc" {
  param "owner" {
    type    = Text
    default = "the Platform team"
  }

  effects = ["fs.write"]

  step "render_header" {
    use  = common.Header
    with = {
      title    = "Onboarding"
      audience = "new hires"
    }
  }

  step "write_onboarding" {
    use  = fs.write
    with = {
      path    = "ONBOARDING.md"
      content = step.render_header.output
    }
  }

  output "onboarding" {
    type = Markdown
    from = step.render_header.output
  }
}

test "team_guide_renders_header_for_team" {
  program = TeamGuide
  with    = { team = "Billing" }

  expect {
    output "guide" contains "Audience: Billing"
    output "guide" contains "Ask the Platform team"
    writes file "GUIDE.md"
    effects = ["fs.write"]
  }
}

test "onboarding_targets_new_hires" {
  program = OnboardingDoc

  expect {
    output "onboarding" contains "new hires"
    writes file "ONBOARDING.md"
    effects = ["fs.write"]
  }
}
```

### What this teaches

- **Shared prompt module** — `prompts.common` is a library other modules depend on.
- **Exported prompt** — `Header` and `Signoff` are `export`ed, so importers can use `common.Header`.
- **Private helper prompt** — `Disclaimer` is _not_ exported; it stays internal to the library. `Footer` shows the same idea locally: a module-local prompt referenced by bare name (`use = Footer`), no alias.
- **Importing with aliases** — `import "./prompts/common.loom" as common`, then `common.Header`.
- **Composing multiple workflows from one library** — `TeamGuide` and `OnboardingDoc` both reuse `common.Header` with different inputs and write different files.
- **Chaining prompt outputs** — `TeamGuide` renders both `Header` and `Footer`, then concatenates their outputs before writing the guide.

## Command sequence

```bash
loom validate examples/prompt-library.loom
loom test examples/prompt-library.loom
loom run examples/prompt-library.loom TeamGuide --team Billing
loom run examples/prompt-library.loom OnboardingDoc
```

## What to commit to Git

- Commit the library and workflow sources: `examples/prompts/common.loom` and `examples/prompt-library.loom`.
- Commit generated docs (`GUIDE.md`, `ONBOARDING.md`) only if you want them tracked; regenerate them in CI to catch drift.
- Keep `.loom/` traces git-ignored.

## How to adapt it

- Grow the library: add more exported prompts to `prompts.common` and import them across many repos.
- Keep helpers private: anything you don't `export` stays internal, so you can refactor it freely without breaking importers.
- Add programs: a new workflow only needs `import "./prompts/common.loom" as common` to reuse everything the library exports.
